# 阶段门控检查清单

> 每个阶段结束前执行。AI 逐项检查，返回 PASS / CONCERNS / FAIL。
> 与"三次确认门控"配合：从 4 个通用维度中选 3 个最相关的，逐维度确认。

## 4 个通用维度

### 1. 正确性 (Correctness)
- [ ] 产出是否符合上一阶段的输入要求？
- [ ] 关键决策是否与 Spec 一致？
- [ ] 数据流/控制流是否有断裂？
- [ ] 边界条件是否覆盖（空/满/异常/超时）？

### 2. 安全性 (Security)
- [ ] `grep -rE "(sk-|api_key|token|password|secret)"` 无硬编码
- [ ] 用户输入是否校验？（类型 + 范围 + 长度）
- [ ] SQL/命令/路径注入风险？
- [ ] 依赖是否有已知 CVE？

### 3. 健壮性 (Robustness)
- [ ] 错误处理是否显式？（不静默吞掉）
- [ ] 日志是否可追溯？（关键路径有 log）
- [ ] 是否有回退方案？（失败能回到已知状态）
- [ ] 外部依赖挂掉后的降级行为？

### 4. 可维护性 (Maintainability)
- [ ] 新增文件在预期范围内？
- [ ] 命名是否符合项目规范？
- [ ] 关键决策是否有注释解释"为什么"？
- [ ] 文件大小是否在 400 行内？

## 阶段特定检查项

### Stage -1~0: 规划层
- [ ] 竞品 ≥3 个，有差异化定位
- [ ] 异常场景 ≥5 个，有应对思路
- [ ] 需求被"反派"挑战过，经过压力测试
- [ ] 输出阶段适用清单（哪些阶段跳、哪些留）

### Stage 1: Spec
- [ ] 功能清单明确 v1 做/不做/v2
- [ ] 验收标准可验证
- [ ] 约束清单完整（技术/时间/人力）

### Stage 2: 技术选型
- [ ] "三问"已答：有现成的吗？一定要自己写吗？选错了怎么办？
- [ ] 核心风险已验证（POC 通过）
- [ ] 最小可行边界明确

### Stage 3: API 设计
- [ ] 每个端点有请求/响应/错误格式
- [ ] 迁移策略已定义（Expand-Contract）
- [ ] 前后端契约双方确认

### Stage 3.5: 故事分解
- [ ] 每个 Story ≤2 文件变更目标
- [ ] Story Context 段包含所有必要链接
- [ ] 依赖关系已标注，无循环依赖
- [ ] sprint.yaml 已更新

### Stage 4~5: 实现层
- [ ] 每个端点 curl 能调通
- [ ] 三态覆盖（loading / empty / error）
- [ ] 代码已自审（AI 先审自己写的）
- [ ] 验收标准逐条通过

### Stage 6~7: 验证层
- [ ] 关键路径测试 ≥5 条
- [ ] 契约验证通过（前后端类型一致）
- [ ] 视觉检查截图通过人工确认

### Stage 8: 预发布（含硬门控）
- [ ] 回滚命令已写并验证
- [ ] 回归测试全绿
- [ ] 冒烟测试通过，观察 ≥15 分钟
- [ ] 🔴 **硬门控** `grep -rE "(sk-|api_key|token|password|secret)"` 无硬编码密钥 — 不可跳过
- [ ] 🔴 **硬门控** 依赖无已知 CVE（`pip audit` / `npm audit`）— 不可跳过
- [ ] 🔴 **硬门控** 注入风险已排除（SQL/命令/路径）— 不可跳过

### Stage 9: 复盘
- [ ] 三问已答：什么做对了？什么造成摩擦？下次怎么改？
- [ ] 教训写入 lessons.md 或 MEMORY.md
- [ ] sprint.yaml 本迭代全部 done

---

## 🔴 硬门控

标记 🔴 的检查项为**硬门控**——不可跳过。用户说"够了进下一步"只跳过普通项，硬门控必须 PASS 才能进下一阶段。目前硬门控集中在 Stage 8（预发布安全），因为 82% 的 AI 代码重大故障源于安全/运行时盲区。

## 判定规则

| 结论 | 含义 | 动作 |
|------|------|------|
| **PASS** | 所有检查项通过（含硬门控） | 进入下一阶段 |
| **CONCERNS** | 有改进建议但非阻塞，硬门控全 PASS | 用户决定：修 or 跳过 |
| **FAIL** | 有阻塞问题 or 硬门控未通过 | 必须修复后重新检查 |

## 使用方式

```
阶段完成 → AI 加载此清单 → 选 3 个最相关维度 → 逐维度检查 → 输出报告
         → 用户确认 → Commit → handoff → /compact → 新会话
```

用户说"够了进下一步" → 直接跳过，不管 CONCERNS。
